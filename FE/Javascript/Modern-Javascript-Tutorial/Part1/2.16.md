## 2.16. 함수 표현식

- 자바스크립트는 함수를 다른 언어처럼 "특정한 동작을 하는 구조"로 취급하지 않고, 특별한 종류의 "값"으로 취급.
- function ~~ 해서 선언할 수도 있지만 `let val = function(){}`식으로 값을 변수에 할당해줄 수도 있음.
- 함수가 만들어진 방식과는 상관없이 함수는 값이므로 변수에 할당 가능. 때문에 alert를 이용해 함수 코드를 출력할 수도 있음.
- `alert( functionName );` 실행 시 함수의 코드를 출력. 자바스크립트는 함수이름 옆에 괄호를 써줘야만 함수가 호출됨. alert에 functionName() 해주면 괄호가 있으므로 함수가 먼저 실행된 후 alert가 실행, 이 때 함수가 반환하는 값을 alert함(리턴이 없으면 undefined).
- 함수는 이름옆에 괄호를 붙여 호출할 수 있다는 점이 일반적인 값과는 다르지만 본질은 값이기 때문에 값에 할 수 있는 일을 함수에도 할 수 있음.
- 함수 선언문에는 세미 콜론 없으나, 함수 표현식에는 끝에 세미콜론이 붙음. `function name(){..}`, `let name = function(){..};`. 함수 표현식은 구문 안에서 값의 역할을 하기 때문에 코드 블록이 아니고 값처럼 취급되어 변수에 할당.

```js
function ask(question, yes, no) {
  if (confirm(question)) yes();
  else no();
}

function showOk() {
  alert("동의하셨습니다.");
}

function showCancel() {
  alert("취소 버튼을 누르셨습니다.");
}

// 사용법: 함수 showOk와 showCancel가 ask 함수의 인수로 전달됨
ask("동의하십니까?", showOk, showCancel);
```

- ask의 인수인 showOk와 showCancel -> 콜백 함수 또는 콜백. 함수를 함수의 인수로 전달, 그 함수를 나중에 호출(called back).

```js
ask(
  "동의하십니까?",
  function () {
    alert("동의하셨습니다.");
  },
  function () {
    alert("취소 버튼을 누르셨습니다.");
  }
);
```

- 위와 같이 이름 없이 선언한 함수-> 익명 함수. 이는 ask 바깥에선 접근 불가.
- 함수는 "동작"을 나타내는 "값". 동작을 대변하는 값인 함수를 변수 간 전달, 동작이 필요할 때 이 값 실행.
- 함수 표현식 vs 함수 선언문 차이
- 1. 문법: 함수 표현식은 표현식이나 구문 구성 내부에 생성, 함수 선언문은 주요 코드 흐름 중간에 독자적인 구문 형태로 존재
- 2. 자바스크립트 언제 함수를 생성하는지: 함수 표현식은 실제 실행 흐름이 해당 함수에 도달했을 때 생성, 함수 선언식은 함수 선언문이 정의되기 전에도 호출 가능. 자바스크립트는 스크립트 실행 전 준비단계에서 전역으로 선언된 함수 선언문을 찾아 해당 함수 생성(초기화 단계).
- 3. 스코프: 엄격모드에서 함수 선언문이 코드블록 내에 위치하면 해당 함수는 블록 내 어디서든 접근 가능하지만 블록 밖에서는 함수에 접근하지 못함.
- 함수 선언문을 통해 함수 선언하는 것을 먼저 고려, 함수 선언문 사용 시 함수 선언 전 호출할 수 있어 코드 구성을 더 자유롭게 할 수 있음. 가독성도 좋음.
